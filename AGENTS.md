# AGENTS.md

## Repository Shape
- Rust workspace crates are `ale-core`, `ale-cli`, and `ale-gui`; edit these root crate directories as source of truth.
- `ale-server` is not in this workspace anymore. Treat `docs/API.md` and any server references as historical unless the task is explicitly about those docs.
- `release/ale-my-eyes-source/` is generated by `scripts/create-release.sh`; do not edit it instead of the root crates.
- `dist/`, `release/`, `target/`, `ale-my-eyes-*` package dirs/archives, model files, and `config/config.json` are build/runtime artifacts.

## Verification Commands
- CI runs `cargo fmt --all -- --check` before `cargo check --workspace` on Linux.
- Focused checks: `cargo check -p ale-core`, `cargo check -p ale-cli`, or `cargo check -p ale-gui`.
- Tests are regular Cargo tests: `cargo test`, `cargo test -p ale-core`, or `cargo test -p ale-core <test_name>`.
- Optional linting is `cargo clippy --workspace --all-targets`; it is not currently a CI gate.
- Run apps with `cargo run -p ale-gui` or `cargo run -p ale-cli -- <transcribe|synthesize|describe|test-connection|status>`.

## Build Notes
- `ale-gui` uses Slint `1.16`; `ale-gui/build.rs` compiles `ale-gui/ui/app.slint`, which imports the other `.slint` files.
- Linux `cargo check --workspace` needs native GUI/audio/capture deps installed: `libspeechd-dev libasound2-dev libfontconfig-dev libpipewire-0.3-dev libwayland-dev libxrandr-dev libdbus-1-dev libegl-dev libgbm-dev libxcb-shape0-dev libxcb-xfixes0-dev`.
- Desktop GUI uses `cpal`/`rodio` audio, `xcap` screen capture, and `enigo` automation; Android uses `oboe`, JNI Camera2, and Slint Android backend.
- `ale-gui` is both a desktop binary (`src/main.rs`) and Android `cdylib` entry (`src/android.rs` via `cargo apk build -p ale-gui --target aarch64-linux-android --lib --release`).

## Architecture Notes
- `ale-core/src/lib.rs` exposes `AleEngine`; default features only enable `cloud`, while `local-inference` gates Whisper/ORT ASR/VLM/LLM and `tts` gates system TTS.
- Long-lived app config is created under the user config directory at `ale-my-eyes/config.json`; tests use `/tmp/ale-my-eyes-test/config.json` or per-test `/tmp` paths.
- `ale-core/src/cloud.rs` is OpenAI-compatible by default (`gpt-4o`, Whisper, TTS); never commit real API keys.
- `ale-gui/src/lib.rs` owns the main app wiring: engine setup, continuous listening, VAD timer, screen/camera capture, settings save, and TTS playback.
- Desktop automation actions are defined in `ale-core/src/actions.rs` and executed by `ale-gui/src/automation.rs`; Android currently reports that desktop automation is unsupported.

## Packaging
- GitHub Actions release flow is the source of truth for published artifacts: tag `v*` or manual dispatch builds Windows `.exe`, Ubuntu `.deb`, and Android `.apk`.
- `./scripts/package-linux.sh` and `./scripts/package-windows.sh` build full release bundles in repo-root `ale-my-eyes-linux/` or `ale-my-eyes-windows/` and generate config examples.
- `./scripts/package-android.sh` requires `ANDROID_NDK_ROOT`, installs `cargo-apk` if missing, and builds both arm64 and armv7 APKs.
- `./scripts/create-release.sh` deletes and recreates `release/`; use it only when intentionally regenerating release snapshots/docs.
